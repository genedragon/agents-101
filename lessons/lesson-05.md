# Lesson 5: Project Kickoff — Design & Scaffold

**Duration:** 60–90 minutes
**Tools used:** Kiro IDE + CLI
**Installs required:** None
**Cost:** $0

---

## Learning objectives

By the end of this session, students will be able to:

1. **Turn a vague idea into a short plan** — what it does, who it's for, what "done enough" looks like
2. **Use the agent as a design partner**, not just a builder — asking it to help think, not just type
3. **Scaffold a project** — get a first working version (even a rough one) running
4. **Keep scope small on purpose** — pick the smallest version that's still cool

---

## What students will produce

- A one-paragraph project plan (problem, user, "done enough")
- A scaffolded project folder with a first version that *runs* (even if ugly/incomplete)
- A short list of "next things to add" for weeks 6–7

---

## Why this lesson matters

This is where the course stops being exercises and becomes *theirs*. The hardest skill isn't coding — it's **scoping**: picking something small enough to finish but real enough to care about. Students who over-scope here stall for the rest of the course. The lesson models "design with the agent, then build the smallest good version." Every student should leave with something that runs.

## Instructor notes (zero-prereq lesson script)

### Idea share + right-sizing (10 min)

Go around: each student says their idea in one sentence. **Your job is scope triage.** For each, quietly sort into:
- ✅ Good size — a single page or small tool
- ⚠️ Too big — help them find the "version 0.1" hiding inside it

> "A whole social network — too big for the club. But 'a page where my friends vote on where to eat' — that's the version-0.1 hiding inside it. Let's build that."

### Core lesson (15 min)

**Two moves to teach:**

**1. Design *with* the agent, not just build with it.**
> "Before you say 'build me X,' ask the agent to *think with you*: 'I want to build [idea]. Ask me 5 questions to help me figure out what it should do.' Let it interview you. You'll end up with a much clearer plan."

**2. Scope to the smallest cool version.**
Put this on the board:
```
What does it DO?  (one sentence)
Who is it FOR?    (a real person, even yourself)
DONE ENOUGH =     (the smallest version you'd be proud to show)
```
> "Done enough is not done perfect. Done enough is 'it works and I'd show a friend.' We add the rest in weeks 6 and 7."

### Demo (10 min)

Live, on the projector, take a sample idea and:
1. Ask the agent to interview you (5 questions).
2. Write the one-paragraph plan together.
3. Ask it to scaffold a first version: *"Make a single-file HTML version of this that runs, even if it's rough."*
4. Open it. It runs. "Ugly is fine. It runs. That's the win today."

### Hands-on (30 min)

Each student:
1. Asks the agent to interview them about their idea.
2. Writes the one-paragraph plan (problem / user / done-enough) in a `PLAN.md`.
3. Asks the agent to scaffold a first version.
4. Gets it to run — however rough.

TA circulates with one question: **"What's the smallest version of this that still runs?"** Rescue anyone spiraling into scope.

### Share (10 min)

Quick round: each student, two sentences — "I'm building X for Y." Save all work (commit if you've shown git, else just save files). Preview: "Next week we put it on the real internet."

## Common pitfalls (instructor cheatsheet)

| Symptom | Cause | Fix |
|---|---|---|
| Idea is way too big | Excitement > scope sense | Find the version-0.1 inside it together |
| "I don't have an idea" | Blank-page freeze | Offer a menu: quiz, tracker, picker, fan page, mini-game |
| Wants it perfect before moving on | Perfectionism | "Done enough today. Polish is weeks 6–7." |
| Agent built something that won't run | Missing file / wrong type | Ask agent: "it doesn't run — read the error and fix it" (previews week 7) |
| Copied a neighbor's idea exactly | Low confidence | Fine — but nudge one personal twist so it's theirs |
