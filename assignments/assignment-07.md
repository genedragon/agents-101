# Assignment 7: Make It Better, Fix What Breaks

**Due:** End of session 7 (finish at home if needed)
**Estimated time:** 25–35 minutes
**Format:** Kiro + your live project. Keep a "bug log."

---

## Goal

Improve your project in small steps, catch the agent when it's wrong, and re-publish a better version.

---

## Part 1 — One improvement at a time (15 min)

Pick **one** thing to improve. Ask the agent for just that one change. Then **check it**:
- Did it do what you asked?
- Did anything else break?

Only when that works, move to the next improvement. Small steps.

---

## Part 2 — Catch a bug, report it well (10 min)

Something will break (that's normal!). When it does, write the agent a **good bug report** using this recipe:

```
1. What I EXPECTED to happen
2. What ACTUALLY happened
3. The exact ERROR message (copy-paste it)
```

Then let it fix the bug. **Verify the fix yourself** — don't just trust that it worked.

Log it:

| Bug | What I expected | What actually happened | How we fixed it |
|---|---|---|---|
| | | | |

---

## Part 3 — Re-publish (5 min)

Push your improved version so the live site updates. Confirm the live URL shows your changes (you may need to refresh / wait a minute).

---

## Part 4 — Was the agent ever confidently wrong? (5 min)

Write 2–3 sentences: did the agent ever *sound* sure but get it wrong today? How did you catch it? This is the most important skill in the whole course.

---

## How this gets graded

"Did you try?" We're looking for:
- At least one real improvement, added in small steps
- A bug log entry with the expected/actual/error detail
- Evidence you **verified** the fix instead of just trusting it

---

## Bring next week

- Your improved, re-published site
- Your bug log
- Curiosity about the advanced track: what would it mean for an agent to *remember you* between sessions?

---

## If you're stuck

If the agent keeps "fixing" the same thing without success, give it the **exact** error text and describe precisely what you expected. If you're truly stuck, roll back to the last version that worked and try a smaller change. Getting stuck and un-stuck *is* the skill — bring your hardest bug to class.
